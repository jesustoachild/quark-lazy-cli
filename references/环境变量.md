# 环境变量

如果安装后的 Skill 目录没有 `.env.local.example`，Agent 应根据本文件创建：

```text
<skills-dir>/quark-lazy-cli/.env
```

创建后必须替换：

- `QAS_HOST`
- `QAS_API_TOKEN`
- 所有 `<skills-dir>` 占位符

完整模板：

```env
# quark-lazy-cli 环境变量配置文件。推荐放在 Skill 目录根部，并通过 --env <skills-dir>/quark-lazy-cli/.env 指定。

# QAS 服务配置
# QAS 后端地址，例如：http://192.168.1.10:15305
QAS_HOST=http://your-qas-host:port
# QAS API Token，可在 QAS Web 管理页面查看
QAS_API_TOKEN=your-qas-api-token

# 系统及搜索策略配置
# LAZY_CLI_EP_SELECTION_POLICY: 选片策略 prefer_quality=最高画质 prefer_size=最大体积
LAZY_CLI_EP_SELECTION_POLICY=prefer_quality
# LAZY_CLI_SEARCH_BATCH_DISP: 搜索结果分页显示数量
LAZY_CLI_SEARCH_BATCH_DISP=30
# LAZY_CLI_SEARCH_EXPIRATION_DAYS: 资源新鲜度阈值（天），超过则视为旧资源，根据剧集更新频率而定，LLM 和 code 顾问，只看新鲜资源
LAZY_CLI_SEARCH_EXPIRATION_DAYS=7
# LAZY_CLI_DRILL_MAX_DEPTH: BFS探查子目录最大深度
LAZY_CLI_DRILL_MAX_DEPTH=3
# LAZY_CLI_MAX_SEARCH_BREADTH: 最大搜索有效资源个数
LAZY_CLI_MAX_SEARCH_BREADTH=10

# Advisor 模式：human（默认）/ code / llm / agent
LAZY_CLI_ADVISOR=code
LAZY_CLI_HUMAN_FILTER=true

# Agent 顾问配置（LAZY_CLI_ADVISOR=agent 时生效）

LAZY_CLI_AGENT_POLL_INTERVAL=15

# LLM 顾问配置（可选，仅 LAZY_CLI_ADVISOR=llm 时生效）
# 支持 OpenAI-compatible API 服务
LAZY_CLI_LLM_OPENAI_API_BASE=https://api.example.com/v1
LAZY_CLI_LLM_OPENAI_API_KEY=your-api-key
LAZY_CLI_LLM_OPENAI_MODEL=your-model-name
LAZY_CLI_LLM_TEMPERATURE=0.4


# 日志配置
LAZY_CLI_DEBUG=false

# 建议使用 Skill 目录下的 runtime 目录，便于 Agent 清理和迁移
LAZY_CLI_LOG_DIR=<skills-dir>/quark-lazy-cli/runtime/logs
LAZY_CLI_REPORT_DIR=<skills-dir>/quark-lazy-cli/runtime/report
LAZY_CLI_AGENT_MSG_DIR=<skills-dir>/quark-lazy-cli/runtime/message
```

环境变量优先顺序：

```text
命令行临时环境变量 > 系统环境变量 > --env 指定的 .env 文件 > 默认值
```
