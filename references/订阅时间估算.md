# 订阅时间估算

本参考用于 Agent 根据 `qslazy task status` 或 `scripts/check_all_status.sh` 的批量输出，推算每部剧/番的最佳订阅更新时间，并写入订阅时间表。

## 什么时候读取

当用户说：

- “帮我接管订阅维护”
- “帮我看看所有剧什么时候更新合适”
- “根据资源发布时间制定订阅时间”
- “生成/更新动漫订阅时间表”

Agent 应读取本参考。

## 批量获取状态

开发目录提供了批量巡检脚本：

```bash
bash scripts/check_all_status.sh
```

它会依次执行：

```bash
qslazy task list
qslazy task status <taskname>
```

输出形态示例：

```text
================================================

  [ 仙剑奇侠传.2025 ] ⏳ Searching for 仙剑奇侠传...

============================================================
  📋 任务：仙剑奇侠传.2025
  💾 网盘：最高 22（更新于：5天前），最低 1，无缺集
  🎯 目标：第22集以上资源
  💎 偏好：最高画质优先

  [2] [5天前 21:41 周二] 仙剑奇侠传叁(2025)4K 臻彩MAX S01E01 - E22 HIFI声 HiveWeb
```

其中 `7` 来自 `LAZY_CLI_SEARCH_EXPIRATION_DAYS`。如果该环境变量改为 `5`，状态页会显示 `5 天内新鲜资源` 和 `5-10 天旧资源`。

## 状态输出解读

`qslazy task status` 会显示两个观察窗口：

- `{LAZY_CLI_SEARCH_EXPIRATION_DAYS}` 天内新鲜资源。
- `{LAZY_CLI_SEARCH_EXPIRATION_DAYS}` 到 `{LAZY_CLI_SEARCH_EXPIRATION_DAYS * 2}` 天旧资源。

列表标记含义：

- `[1]`、`[2]`、`[3]`：验证有效资源，可用于判断可下载资源和发布时间。
- `[X]`：验证失效资源，仅用于观察发布规律，不可选择，不进入 advisor/BFS/下载流程。

失效资源仍然有参考价值：

- 如果同一资源站每天都发布但很快失效，说明它发布活跃但稳定性差。
- 如果新鲜资源大多失效，而较晚发布的资源有效，订阅时间应更偏向画质优先或延后执行。
- 如果旧资源持续失效，说明该源不适合作为长期订阅依据。

注意：`update` 的 human 选择界面也会展示 `[X]` 失效资源，但 `[X]` 不占数字序号，不能被选择；LLM advisor 只接收有效资源列表，并只看到失效资源摘要。

## 估算规则

Agent 从每个任务的候选资源发布时间里提取：

- 最早有效资源发布时间
- 最后一批高质量资源发布时间
- 失效资源发布时间和数量
- 资源是否集中在同一天、同一时间段
- 是否有旧资源或失效资源干扰
- 网盘当前最高集和更新时间

然后先问用户偏好：

```text
你希望订阅更新时间偏向哪种策略？
1. 时间优先：按最早有效资源发布时间 + 4 小时安排，尽快看到更新。
2. 画质优先：按最后一批高质量资源发布时间 + 4 小时安排，等待高码率、HDR/DV、字幕组或更完整资源。
```

默认建议：

- 用户不明确选择时，使用画质优先。
- 对经常延迟或资源波动大的剧，加一次次日上午兜底重试。
- 对已下线或长期无新资源的剧，标为手动维护。
- 如果新鲜资源中 `[X]` 很多，但有效资源集中在更晚时间，优先参考有效资源发布时间，不要按失效资源的最早时间提前订阅。
- 如果 `[X]` 连续多周出现在同一发布源，可在备注中标记“该源易失效”，避免把它作为唯一订阅依据。

## 示例推导

### 仙剑奇侠传.2025

候选资源：

- 周二 21:55：4K 臻彩MAX，更新至 22 集
- 周二 21:41：4K 臻彩MAX，S01E01 - E22
- 周二 15:13：4K，更新至 22 集

建议：

- 时间优先：周二 19:13 以后。
- 画质优先：周三 01:55 左右。
- 如果用户偏好稳定高质量资源，建议周三 02:00。

### 斗破苍穹

候选资源：

- 周六 18:48 到 20:23 集中发布。
- 多个 4K / 10bit / 臻彩资源，更新至 199 集。

建议：

- 时间优先：周六 22:50 左右。
- 画质优先：周日 00:30 左右。
- 若用户不急，建议周日 00:30 或周日 01:00。

## 订阅时间表模板

| taskname | 剧名/显示名 | 更新策略 | 建议订阅更新时间 | 定时表达式 | 状态 | 备注 |
|----------|-------------|----------|------------------|-------------|------|------|
| 剧名1 | 剧名1 | 画质优先 | 周六 02:30 | `30 2 * * 6` | 正常 | 参看：剧名1 订阅特殊说明 |
| 剧名2 | 剧名2 | 时间优先 | 周日 00:30 | `30 0 * * 0` | 正常 | |
| 剧名3 | 剧名3 | 手动维护 | — | ❌ 已下线 | 暂停 | 资源下线或订阅暂停 |

字段说明：

- `taskname`：QAS 里的订阅任务名，也是 qslazy 命令参数。
- `剧名/显示名`：给用户看的名称，可以比 `taskname` 更自然。
- `更新策略`：`时间优先`、`画质优先`、`手动维护` 或用户指定策略。
- `建议订阅更新时间`：自然语言时间，方便用户阅读。
- `定时表达式`：OpenClaw 使用的 cron 格式时间表达式。
- `状态`：`正常`、`暂停`、`手动维护`、`待确认` 等。
- `备注`：特殊资源规则、用户偏好、下线原因或参考说明。

## 订阅特殊说明示例

### 剧名1  订阅特殊说明

- 目前资源发布有 2 套剧集编号格式：
  - A 编号格式：百炼成神第三季从 01 开始，剧集名特征是 2 位数字，例如“更新至 22 集”。
  - B 编号格式：百炼成神从第一季到现在一直使用累计编号，例如 `EP127`。

- 用户需求：
  - 选择 A 编号格式的 URL。
  - 使用 A 编号格式 URL 中的最新剧集编号。

推荐执行方式：

```bash
LAZY_CLI_ADVISOR=llm qslazy update new 剧名1  \
  --env <skills-dir>/quark-lazy-cli/.env \
  --add-prompt "目前资源发布有两套剧集编号格式：A 格式为第三季从 01 开始的两位数字编号，如更新至 22 集；B 格式为从第一季累计到现在的 EP1xx 编号，如 EP127。请选择 A 格式的 URL，并取 A 格式中的最新剧集编号。"
```

## Agent 维护规则

- 新增剧名前，先运行 `qslazy task status <taskname> --env <skills-dir>/quark-lazy-cli/.env`。
- 根据 stdout 中的资源发布时间推算更新时间，不要凭空填写。
- 配置定时任务前，必须向用户确认更新策略。
- 已下线、资源不稳定或需要人工判断的剧，状态标记为 `手动维护` 或 `待确认`。
- 特殊资源规则不要塞进主表，放到下方“订阅特殊说明”。
